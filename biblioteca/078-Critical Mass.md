# [[Critical Mass]]

> **Teste de Pertinência:** *Você está tentando alcançar ponto onde mudança se torna auto-sustentável, ou está tentando identificar quantidade mínima necessária antes que sistema mude de estado? Se a resposta for "Sim", este modelo é a ferramenta correta para o momento.*

## 1. Definição e Mecanismo
**O que é:**
Critical Mass (Massa Crítica) refere-se a tamanho mínimo ou quantidade de fator contribuinte (pessoas, recursos, influência) necessária antes que mudança se torne auto-sustentável. Uma vez que massa crítica é alcançada, momentum se constrói por si só. Tipping Point (Ponto de Virada) é threshold além do qual sistema muda de um estado para outro rapidamente e frequentemente irreversivelmente. É quando mudanças acumuladas empurram sistema além de boundary crítico.

**Explicação:**
Este modelo funciona porque reconhece que sistemas frequentemente têm thresholds - pontos além dos quais comportamento muda dramaticamente. Antes de alcançar massa crítica, mudanças são lentas e podem não se sustentar. Após alcançar massa crítica, feedback positivo acelera mudança e sistema pode mudar rapidamente para novo estado. Isto significa que pequenas mudanças antes de threshold podem ter pouco efeito, mas após threshold, mesmo pequenas mudanças adicionais podem ter grandes efeitos.

A lógica central é que através de acumulação de fatores contribuintes (pessoas, recursos, influência), sistema se aproxima de threshold. Uma vez que threshold é cruzado, mudança acelera devido a feedback positivo. Sistema pode então transicionar para novo equilíbrio rapidamente, e pode não reverter facilmente para estado anterior mesmo se pressões diminuem, devido a histerese, inércia ou loops de feedback reforçadores.

Funciona porque:
- **Reconhece thresholds:** Sistemas têm thresholds além dos quais comportamento muda dramaticamente.
- **Cria feedback positivo:** Após threshold, feedback positivo acelera mudança.
- **Permite mudança rápida:** Uma vez que threshold é cruzado, mudança pode ser rápida e dramática.
- **Pode ser irreversível:** Mudança pode não reverter facilmente após threshold ser cruzado.

**Exemplos empíricos:**
- **Mudança de normas sociais (~25%):** Quando minoria comprometida alcança cerca de 25% de população, pode confiavelmente derrubar norma existente.
- **Resistência não violenta (3.5%):** Movimentos não violentos disciplinados e sustentados têm sucesso quando cerca de 3.5% da população se junta.

## 2. Pressupostos Fundamentais
*O modelo só é válido se estas premissas forem verdadeiras:*
* **Pressuposto 1:** Sistemas têm thresholds além dos quais comportamento muda dramaticamente.
* **Pressuposto 2:** Antes de alcançar massa crítica, mudanças são lentas e podem não se sustentar.
* **Pressuposto 3:** Após alcançar massa crítica, feedback positivo acelera mudança.
* **Pressuposto 4:** Threshold específico depende de estrutura do sistema, força de resistência, visibilidade, contexto.

## 3. Contexto de Aplicação

### Quando Usar (Luz Verde)
* Situações onde o modelo produz boas decisões rapidamente:*
- **Mudança social:** Ao tentar criar mudança social, identificar massa crítica necessária para mudança se tornar auto-sustentável.
- **Adoção de tecnologia:** Ao tentar acelerar adoção de tecnologia, identificar massa crítica de usuários necessária.
- **Mudança organizacional:** Ao gerenciar mudança organizacional, identificar massa crítica de apoio necessária.
- **Inovação:** Ao inovar, identificar massa crítica de adoção necessária para inovação se tornar auto-sustentável.
- **Entender transições:** Ao entender transições de sistema, reconhecer papel de massa crítica e tipping points.

**Domínios Típicos:** Mudança Social, Adoção de Tecnologia, Mudança Organizacional, Inovação, Sistemas.

### Quando Evitar (Luz Vermelha)
* Situações onde o modelo induz ao erro ou simplifica demais a realidade:*
- **Quando não há threshold:** Se sistema não tem threshold claro ou mudança é linear, modelo pode não ser útil.
- **Quando threshold é muito alto:** Se threshold é tão alto que é impraticável alcançar, modelo pode não ser útil.
- **Não como desculpa para não tentar:** Não usar modelo como desculpa para não tentar mudança se threshold parece alto.
- **Não assumir que sempre existe:** Não assumir que massa crítica sempre existe - alguns sistemas podem não ter threshold claro.

## 4. Zona de Risco (Diagnóstico)

### Uso Incorreto Comum
* Como amadores costumam distorcer esse modelo:*
- **Desistir muito cedo:** Desistir antes de alcançar massa crítica porque mudança parece lenta.
- **Não reconhecer quando alcançou:** Não reconhecer quando massa crítica foi alcançada e mudança está acelerando.
- **Assumir threshold fixo:** Assumir que threshold é fixo quando pode variar dependendo de fatores.
- **Ignorar fatores que afetam threshold:** Ignorar que estrutura do sistema, força de resistência, visibilidade afetam onde threshold está.

### Sinais de que Você Saiu do Modelo
* **Indicador Comportamental:** 
  - Você está desistindo antes de alcançar massa crítica.
  - Você não está reconhecendo quando massa crítica foi alcançada.
  - Você está assumindo que threshold é fixo.
  - Você não está considerando fatores que afetam threshold.

* **Evidência Prática:** 
  - Você desiste de mudança antes de alcançar massa crítica.
  - Você não aproveita quando massa crítica é alcançada e mudança acelera.

## 5. Passos de Pensamento (Algoritmo)
1. **Identifique sistema e mudança desejada:** Identifique sistema e mudança que você está tentando criar.

2. **Estime massa crítica:** Estime massa crítica necessária - quantidade mínima de fator contribuinte (pessoas, recursos, influência) necessária antes que mudança se torne auto-sustentável.

3. **Considere fatores que afetam threshold:** Considere fatores que afetam onde threshold está:
   - Estrutura do sistema / efeitos de rede
   - Força de resistência ou inércia
   - Visibilidade / feedback
   - Contexto / incentivos

4. **Monitore acumulação:** Monitore acumulação de fatores contribuintes - você está se aproximando de threshold?

5. **Reconheça fase de acumulação lenta:** Reconheça que antes de massa crítica, mudanças podem ser lentas - não desista muito cedo.

6. **Identifique quando threshold é cruzado:** Identifique quando threshold é cruzado - mudança começa a acelerar? Feedback positivo está operando?

7. **Aproveite aceleração:** Após threshold ser cruzado, aproveite aceleração - feedback positivo pode criar mudança rápida.

8. **Reconheça possível irreversibilidade:** Reconheça que após threshold ser cruzado, mudança pode não reverter facilmente.

9. **Monitore e ajuste:** Após aplicar modelo, monitore se está funcionando como esperado. Ajuste estimativas de massa crítica se necessário.

10. **Use para planejamento:** Use modelo para planejar - quanto esforço é necessário para alcançar massa crítica? Que fatores podem reduzir threshold?

* **Erro:** Ignorei este modelo ao tentar criar mudança e paguei o preço ao desistir antes de alcançar massa crítica, resultando em mudança que não se sustentou.
